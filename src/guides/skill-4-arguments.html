Intro to Command Arguments

Most commands need to know what to act on. This "what" is called an argument.

What Are Arguments?

An argument tells the command what file, folder, or text to work with:

cat notes.txt

notes.txt is the argument — the file to display.

cd Documents

Documents is the argument — the folder to enter.

Commands Without Arguments

Some commands work fine without arguments — they have sensible defaults:

ls

Lists the current directory (no argument needed).

pwd

Shows current location (never needs an argument).

Commands With Arguments

Adding an argument changes what the command operates on:

ls Documents

Lists the Documents folder instead of the current directory.

cat ~/.zshrc

Displays your shell configuration file.

Multiple Arguments

Many commands accept multiple arguments:

cat file1.txt file2.txt

Displays both files, one after the other.

ls Documents Downloads

Lists both folders.

Paths as Arguments

Arguments are often file or folder paths:

  • file.txt — file in current directory
  • Documents/file.txt — relative path
  • /Users/you/file.txt — absolute path (starts with /)
  • ~/file.txt — in your home folder (~ means home)

Practice

  1. ls — list current directory
  2. ls ~ — list your home folder
  3. cat ~/.zshrc — view your shell config
  4. echo Hello World — echo takes text as arguments
