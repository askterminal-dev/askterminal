Best Practices

Get better results from Claude Code by following these patterns.

Be Specific

Good prompts include context:

  • Less helpful: "Fix the bug"
  • More helpful: "Fix the null pointer error in UserService.getProfile()"

Mention file names, function names, error messages, and expected behavior.

Review Changes Before Accepting

When Claude shows you a diff (the changes it wants to make):

  • Read through the changes
  • Make sure they match what you asked for
  • Check that nothing unexpected is being modified

It's faster to catch issues before accepting than to fix them after.

Iterate and Refine

If the first result isn't perfect, guide Claude:

  • "Good, but use async/await instead of callbacks"
  • "Can you make this more concise?"
  • "Actually, put that in a separate file"

Claude learns from your feedback within the conversation.

Use Git Commits as Checkpoints

Create commits between major changes:

  1. Ask Claude to make a change
  2. Review and test it
  3. Commit if it works
  4. Move to the next change

This gives you safe points to return to.

Let Claude Explore First

For unfamiliar codebases, ask questions before requesting changes:

  • "How does authentication work in this app?"
  • "What would be the best place to add this feature?"
  • "Are there existing patterns I should follow?"

Break Down Large Tasks

Instead of "Rewrite the entire auth system":

  1. "Explain the current auth flow"
  2. "What are the problems with it?"
  3. "Design a better approach"
  4. "Implement step 1 of the new design"

Trust But Verify

Claude is helpful but not perfect. Always:
