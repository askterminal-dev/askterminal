Understanding Permissions

Claude Code asks for permission before making changes. This keeps you in control of what happens to your code.

The Permission Prompt

When Claude wants to do something that changes files or runs commands, you'll see a prompt asking for approval. You can:

  • Allow — Let Claude proceed with this action
  • Deny — Stop this action from happening
  • Allow All — Trust this type of action for the session

What Requires Permission

Writing or editing files
Running terminal commands
Commands that could have wide impact (like rm -rf)

What Doesn't Require Permission

Reading files
Searching for files or content
Looking up documentation online

Read-only operations are generally allowed without prompting.

When to Deny

Say no when:

  • You want to review the change first
  • The action doesn't match what you asked for
  • You want to try a different approach
  • You're not sure what the command does

It's always okay to deny and ask Claude to explain first.

Trusting Actions

If you're doing repetitive work, you can allow entire categories:

  • "Allow all file edits" — Useful when refactoring
  • "Allow all bash commands" — Useful for build/test cycles

These permissions last only for your current session.

Safety Net: Git

Always work in a git repository. If something goes wrong:

git diff

See what changed

git checkout .

Undo all uncommitted changes

Commit before big changes so you have a restore point.
